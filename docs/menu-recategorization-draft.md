## 전제 / 안전성

- 카테고리는 `MenuItem.id`에 인코딩됨 (`id / 1000 == 카테고리번호`). `placement`(정렬값)도 동일.
- 주문/리포트는 메뉴를 **이름(name) 기준**으로만 저장·집계함 → **ID 재배치는 과거 데이터에 영향 없음.** 이름만 바꾸지 않으면 안전.
- 따라서 개편 = ① `MenuCategories.names` 배열 교체 + ② Firestore `menu/current`의 각 항목 `id`·`placement` 재배치(이름·가격·inuse 그대로).

## Benn 보정 반영 내역 (1차)

- 녹차라떼 / 초코라떼 / 생강라떼 / 아몬드크림라떼 → **커피 계열로 HOT/ICE커피 유지**
- 복숭아·레몬 아이스티 → **"차"로 유지**
- 과일사이다(4종) · 분다버그(3종) → **"기타"로 이동**
- `ㄴ 아이스컵`(기존 4017) → 기존 ID상 분다버그 바로 뒤 옵션이라 **분다버그 따라 "기타"로 이동** (🚩 확인 부탁)
