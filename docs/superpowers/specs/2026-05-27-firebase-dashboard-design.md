# Firebase 매출 대시보드 — 설계 문서

작성일: 2026-05-27
브랜치: v3

## 1. 배경 & 목적

HolyBean은 교회 카페용 안드로이드 POS이며, v3에서 백엔드를 AWS → Firebase/Firestore로 이전했다. Firestore에는 이미 날짜별 매출 집계(`reportRollups/{date}`)가 쌓이고 있다.

이 프로젝트는 **카페 운영자(비기술 사용자)가 POS 기기 없이도 어디서나 매출을 확인하고 엑셀로 내려받을 수 있는 웹 대시보드**를 만든다. 앱 내 리포트 기능의 웹 확장판 성격이다.

### 주 사용자
- 카페 운영자 1인. 비기술 사용자. 폰과 PC를 모두 쓴다.

### 핵심 가치
- 하루를 골라 그날의 **총매출 · 전체 잔 수 · 메뉴별 판매 내역**을 본다.
- 그날 데이터를 **엑셀(.xlsx)로 내려받는다.**

## 2. 범위

### 포함
- 이메일/비밀번호 로그인
- 날짜 선택(실제 매출 기록이 있는 날끼리 ◀▶ 이동)
- 그날의 총매출, 전체 잔 수, 판매 내역(개수 내림차순)
- 현재 보고 있는 날의 엑셀 내려받기

### 제외 (추후 확장 여지)
- 기간 선택 / 매출 추이 그래프
- 결제수단 비율
- 미결제 외상 현황
- 데이터 쓰기/수정 기능 (읽기 전용 대시보드)

## 3. 아키텍처 & 기술 스택

**순수 정적 SPA, 빌드 단계 없음.** 프레임워크/번들러를 쓰지 않는다.

```
[브라우저]
  │ Firebase Auth (이메일/비밀번호 로그인)
  │ Firestore Web SDK (modular, CDN) ──▶ reportRollups/{date} 읽기
  │ SheetJS (xlsx, CDN) ──▶ .xlsx 생성·다운로드
  ▼
[Firebase Hosting] ← 정적 파일 (index.html / app.js / styles.css)
```

- **호스팅**: Firebase Hosting. 기존 `firebase.json`에 `hosting` 설정 추가, 정적 디렉터리 배포.
- **보안**: 기존 `firestore.rules` 그대로 재사용 (`read: if signedIn()`). 서버 코드 0.
- **외부 의존성**: Firebase JS SDK(modular, CDN ESM), SheetJS(CDN), Pretendard 웹폰트(CDN). 빌드 도구·npm 의존성 없음.

### 파일 구성 (예시)
- `dashboard/index.html` — 마크업 + CDN 링크
- `dashboard/app.js` — Auth, Firestore 읽기, 렌더링, 엑셀 export (ES module)
- `dashboard/styles.css` — 스타일

(정확한 디렉터리 위치는 구현 계획에서 기존 `firebase/` 구조와 조율해 결정한다.)

## 4. 화면 구성 (단일 스크롤, 반응형)

폰에서는 꽉 차게, PC에서는 가운데 정렬(`max-width` 약 520px, 좌우 여백). 동일 코드로 반응형 처리.

### 4.1 로그인 화면
- 브랜드 라벨 "HolyBean", 제목 "매출 대시보드"
- 이메일 / 비밀번호 입력, "로그인" 버튼
- 로그인 성공 시 대시보드 표시. 실패 시 에러 메시지.

### 4.2 대시보드
1. **상단바**: 브랜드 라벨 + "로그아웃"
2. **날짜 헤더**: `5월 27일 (화)` 형식 + ◀▶ 버튼
3. **통계 카드 2개**: 이날 총매출(원), 전체 잔 수(잔)
4. **"엑셀로 내려받기" 버튼**
5. **판매 내역 (많이 팔린 순)**: 그날 팔린 모든 메뉴를 개수 내림차순으로 — 메뉴명 · 개수 · 금액

## 5. 데이터 매핑 (`reportRollups/{date}`)

문서 구조 (기존 `firebase/src/rebuild.ts` 기준):
```ts
reportRollups/{YYYY-MM-DD} = {
  menuSales: { [메뉴명]: { quantity: number, sales: number } },
  paymentSales: { [결제수단]: number },
  total: number,
}
```

- **이날 총매출** = `total`
- **전체 잔 수** = `menuSales` 전 항목의 `quantity` 합
- **판매 내역** = `menuSales`를 `{ name, quantity, sales }` 배열로 펴서 `quantity` 내림차순 정렬
- `paymentSales`는 이번 범위에서 사용하지 않음

## 6. "실제 있는 날만" 날짜 이동

로그인 후 `reportRollups` 전체를 **문서 ID(`YYYY-MM-DD`) 오름차순으로 한 번 로드**해 메모리 배열에 담고, ◀▶는 그 배열의 인덱스를 ±1 이동한다. 양 끝(가장 최근/가장 오래된)에서는 해당 화살표를 비활성화한다.

- **최초 진입**: `getDocs(query(reportRollups, orderBy(documentId(), "asc")))` → `[{date, data}]` 오름차순 배열, 초기 인덱스 = 마지막(가장 최근 날)
- **◀ / ▶**: 인덱스 −1 / +1 (경계에서 비활성). 추가 읽기 없음 — 렌더는 동기 함수.

> **설계 변경 이력:** 초기 설계는 "이동당 1건만 읽기"를 위해 `documentId()` 범위 쿼리 + `orderBy(documentId(), "desc")`를 쓰려 했으나, 구현 중 에뮬레이터 검증에서 Firestore가 **문서 키 내림차순 정렬("descending key scan")을 지원하지 않음**을 확인했다(2026-05-27). 오름차순 키 스캔은 지원되므로 전체를 한 번 오름차순 로드하는 방식으로 변경했다. 이 카페는 단일 사용자이고 운영일(=문서) 수가 수백 건 규모라 1회 로드(수백 reads, 무료 한도 내)는 충분하며, 부수적으로 렌더 시 경계 판별 읽기와 비동기 race가 제거된다. 추가 인덱스(`firestore.indexes.json`)는 단일 필드 키 정렬이라 불필요하다.

## 7. 엑셀 Export

- 현재 보고 있는 날의 판매 내역으로 워크북 생성 (클라이언트 전용, 서버 호출 없음).
- 시트 내용: 헤더 `[메뉴, 개수, 금액]` + 메뉴 행들(개수 내림차순) + 합계 행 `[합계, 전체 잔 수, 총매출]`.
- 파일명: `holybean-{YYYY-MM-DD}.xlsx`
- SheetJS `aoa_to_sheet` → `writeFile`.

## 8. 인증 & 보안

- **Firebase Auth 이메일/비밀번호.** 단일 공용 계정을 Firebase 콘솔에서 미리 생성해 운영자에게 전달.
- 미로그인 상태에서는 대시보드 접근 불가 (로그인 화면 표시). `onAuthStateChanged`로 세션 유지.
- Firestore 읽기는 기존 규칙(`read: if signedIn()`)으로 보호됨. 대시보드는 읽기만 하며 쓰기/삭제하지 않는다.

## 9. 비주얼 가이드

- **흰 배경 미니멀.** 화려함보다 가독성·직관성.
- 폰트: Pretendard (시스템 산세리프 폴백).
- 색: 모노톤(잉크 `#1b1b1b`, 회색 `#8a8a8a`, 라인 `#ececec`) + 따뜻한 캐러멜 강조색 `#b5651d` 한 점(강조 숫자·export 버튼).
- 숫자는 큰 크기 + `tabular-nums`.
- 넉넉한 여백, 부드러운 라운드 카드.

## 10. 엣지 케이스 & 에러 처리

- **데이터 전혀 없음**: 최초 쿼리 결과가 비면 "아직 매출 기록이 없어요" 빈 상태 표시.
- **가장 최근/가장 오래된 날**: 해당 방향 화살표 비활성.
- **Firestore 읽기 실패**: 사용자에게 에러 메시지 표시 (조용히 빈 화면 X).
- **로그인 실패**: 입력 아래 에러 메시지.
- **메뉴 0건인 날**: 통계는 0, 판매 내역은 빈 상태.

## 11. 검증

빌드/프레임워크가 없으므로 수동 검증 중심:
- Firebase 에뮬레이터(기존 `firebase/` 설정)에 샘플 `reportRollups` 데이터를 넣고 로그인 → 날짜 이동 → 엑셀 다운로드를 직접 확인.
- 날짜 경계(최신/최초)에서 화살표 비활성 확인.
- 데이터 없음 / 읽기 실패 빈 상태 확인.
- 폰·PC 폭에서 반응형 레이아웃 확인.
